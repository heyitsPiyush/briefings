Hardware Hacking

deep

Recent hardware security research reveals a critical, unpatchable BootROM vulnerability, dubbed "usbliter8," affecting Apple's A12 and A13 chips, as well as the S4 and S5 series found in devices like the iPhone XS through 11 series and various iPads and Apple Watch models. This hardware flaw in the Synopsys DWC2 USB controller permits arbitrary code execution by manipulating an internal hardware pointer through specially crafted USB packets in Device Firmware Update (DFU) mode, allowing data writes to protected memory regions before the signed boot chain loads. The exploit requires physical access via a dedicated RP2350-based microcontroller board, taking less than two seconds to complete. Additionally, a critical firmware authentication bypass vulnerability, CVE-2026-35075, has been identified in various devices using hard-coded passwords, allowing unauthenticated remote attackers to gain full administrative access by extracting default credentials directly from firmware images.

In the realm of embedded security and IoT, a significant BootROM vulnerability, CVE-2026-25262, has been discovered in several Qualcomm chip series, including MDM9x07, MDM9x45, MDM9x65, MSM8909, MSM8916, MSM8952, and SDX50, impacting IoT devices, industrial equipment, smartphones, and automotive control units. This flaw, discovered by Kaspersky ICS CERT, allows for malicious code injection at the BootROM level, compromising the device before the operating system loads. Furthermore, a new fault injection attack framework named "InjectV" has been introduced for RISC-V platforms, built on the gem5 simulator, to assess hardware security by systematically injecting transient faults in registers and memory, particularly at security-critical execution points like control-flow decisions. Side-channel attacks continue to evolve, with research demonstrating cross-VM side-channel attacks by exploiting Intel's Data Streaming Accelerator (DSA) on Xeon processors, bypassing typical I/O virtualization protections through the Device TLB and shared work queue. The IoT landscape faces increasing threats from botnet recruitment and supply chain compromises, with reports highlighting botnets like Aisuru/TurboMirai achieving significant DDoS capabilities and "BadBox 2.0" pre-infecting millions of devices.
