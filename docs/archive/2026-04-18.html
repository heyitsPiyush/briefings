Hardware Hacking

deep

In recent hardware and embedded security news, firmware vulnerabilities remain a critical concern, with a stack-based buffer overflow (CVE-2026-3044) identified in Tenda AC8 router firmware version 16.03.34.06. This flaw in the webCgiGetUploadFile function of the Httpd Service component allows remote attackers to execute arbitrary code by manipulating the boundary argument. Additionally, Anviz CX2 Lite and CX7 devices are susceptible to unauthenticated firmware uploads, enabling attackers to plant and execute malicious code, and achieve remote code execution through unverified update packages (CVE-2026-40066, CVE-2026-35546). These instances highlight the persistent danger of insecure firmware and the necessity for robust input validation and authenticated update mechanisms across diverse embedded systems.

The embedded security landscape also reveals a heightened focus on side-channel attacks and supply chain integrity. Research into side-channel attack mitigation for quantum-resistant MCP metadata demonstrates that even post-quantum cryptography implementations on AI chips can leak secret keys through electromagnetic radiation and power consumption, as shown by attacks like BarraCUDA on NVIDIA Jetson chips. Furthermore, the supply chain continues to be a major attack vector, with incidents such as the CPUID breach distributing the STX RAT via trojanized CPU-Z and HWMonitor downloads. New hardware hacking tools like the Flipper One, a pocket-sized Linux computer, and DIY projects like HaleHound, a multi-protocol offensive security toolkit with WiFi, Bluetooth, SubGHz, 2.4GHz, and RFID/NFC support, are emerging, offering advanced capabilities for security research and penetration testing.
