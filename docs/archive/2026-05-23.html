Hardware Hacking

deep

Recent hardware hacking and embedded security news highlights several critical vulnerabilities and evolving attack vectors. Ubiquiti recently patched three maximum severity vulnerabilities in UniFi OS, including CVE-2026-34908 (improper access control), CVE-2026-34909 (path traversal), and CVE-2026-34910 (command injection), which could allow remote attackers to make unauthorized changes or access system files on UniFi Consoles and applications. Additionally, a cross-site scripting (XSS) vulnerability, CVE-2026-25789, was discovered in the firmware update page of Siemens devices, allowing remote attackers to execute JavaScript through malicious filenames. Qualcomm also addressed several critical and high-impact vulnerabilities in its May 2026 Security Bulletin, affecting areas like Qualcomm Software Center and Boot processes.

IoT security remains a significant concern, with over 50% of connected devices containing critical vulnerabilities that attackers can exploit immediately, and routers accounting for over 75% of all IoT-related cyberattacks. In RF hacking, the Flipper Zero, Proxmark3, and HackRF One continue to be prominent tools for RFID/NFC testing, radio and wireless signal research, and Wi-Fi auditing, demonstrating capabilities in cloning cards, recording and simulating infrared signals, and generating rogue access points. Supply chain security is also a major focus, with attacks targeting trusted components to infiltrate systems indirectly, as seen in the Foxconn Nitrogen ransomware incident where 8 terabytes of data, including architectural network topology maps for companies like Apple, Google, Nvidia, Dell, and Intel, were exfiltrated. New tools like Kali Linux 2026.1 introduce capabilities such as XSStrike for XSS vulnerabilities and SSTImap for Server-Side Template Injection, further assisting security professionals in identifying weaknesses.

Exploit Development

deep

Latest exploit development news as of May 23, 2026, includes several critical vulnerabilities and public proof-of-concept (PoC) releases across Windows, Linux, and browser platforms. A significant Windows local privilege escalation (LPE) zero-day, dubbed "MiniPlasma," affecting fully patched Windows 11 systems, leverages a re-emerged vulnerability (CVE-2020-17103) in the Cloud Filter driver (cldflt.sys), allowing standard users to gain SYSTEM privileges through registry key manipulation via the CfAbortHydration API. Additionally, another Windows BitLocker bypass, "YellowKey" (CVE-2026-45585), with public PoC, allows physical attackers to sidestep encryption by abusing the Windows Recovery Environment (WinRE) during pre-boot to gain unrestricted shell access. Multiple critical Chrome browser vulnerabilities were patched, including a use-after-free in WebRTC (CVE-2026-9111) enabling remote code execution on Linux systems and a UI spoofing flaw (CVE-2026-9110) on Windows. Separately, a Chromium bug that can persistently execute JavaScript across browser restarts was publicly leaked and remains unpatched.

Linux kernel exploits have seen a surge in public PoC releases, primarily focusing on page-cache corruption issues from "zero-copy" functionality. This includes "CopyFail" (CVE-2026-31431), "Dirty Frag" (CVE-2026-43284, CVE-2026-43500), and "Fragnesia" (CVE-2026-46300), all of which allow unprivileged local attackers to achieve root privileges. These vulnerabilities often abuse trusted syscalls like splice and vmsplice. Another nine-year-old Linux kernel flaw, CVE-2026-46333 (codenamed "ssh-keysign-pwn"), enables local users to disclose sensitive files and execute arbitrary commands as root on major distributions. Notably, security researchers leveraged Anthropic's Mythos Preview AI model to develop the first public macOS kernel memory corruption exploit on M5 silicon, bypassing Apple's Memory Integrity Enforcement (MIE) in just five days, demonstrating advanced AI capabilities in exploit development. AI × Cybersecurity

deep

AI-powered cyberattacks are rapidly escalating in sophistication and speed as of May 2026, with attackers leveraging AI to automate phishing, create realistic impersonations, and generate zero-day exploits. Google notably identified the first AI-developed zero-day exploit designed to bypass two-factor authentication, highlighting a significant shift in threat capabilities. The financial sector and critical infrastructure are particularly targeted, facing risks like those exposed by Anthropic's Mythos AI model, necessitating urgent patching efforts by institutions globally. IBM is strengthening its security portfolio through Project Glasswing, an initiative with Anthropic, Amazon Web Services, Apple, and others, focused on using frontier AI models to identify software vulnerabilities and accelerate defensive responses.

Prompt injection remains a critical vulnerability, with OWASP ranking it as the number one risk (LLM01:2025) for AI applications. Recent research from Moonlock and presentations at the RSA Conference 2026 demonstrated that Apple Intelligence can be manipulated through prompt injection, including indirect methods hidden in malicious emails or web content. To counter these threats, several defensive AI tools are emerging, such as Llama Guard 3 for content filtering, SentinelSphere 2.1 for real-time agent threat detection, and Microsoft PyRIT combined with OWASP LLM Top 10 v2 for prompt defense and threat modeling. Securing AI pipelines and Retrieval-Augmented Generation (RAG) systems is also gaining traction, with the NSA's Artificial Intelligence Security Center (AISC) releasing guidance on the Model Context Protocol (MCP) to address risks like serialization and agent misuse in AI-driven automation.
