Hardware Hacking

deep

Recent developments in embedded security highlight a critical focus on firmware vulnerabilities and the increasing sophistication of fault injection attacks. Six new U-Boot flaws were discovered by Binarly researchers, impacting devices from home routers to data-center server management chips, with two of these bugs allowing malicious code execution before bootloader signature verification. Additionally, Marlin Firmware through version 2.1.2.7 was found to have an out-of-bounds write vulnerability in its M421 G-code handler, enabling corruption of firmware memory and denial-of-service or state corruption via crafted G-code commands. Researchers have also introduced InjectV, a fault injection attack framework for RISC-V platforms, built on the gem5 simulator, capable of systematically assessing vulnerabilities at security-critical execution points and achieving significant time savings over traditional physical fault experiments.

The landscape of IoT and supply chain security is rapidly evolving, with a growing emphasis on hardware-level protections and the risks posed by interconnected systems. NIST's recent draft report, IR 8320E, details approaches to confidential computing for data in cloud workloads, aiming to protect data during processing in memory, especially for AI workloads on cloud infrastructures. The proliferation of IoT devices and embedded processors, including those in Raspberry Pi and NVIDIA Jetson, has driven the adoption of secure boot and hardware Roots of Trust from high-end devices to microcontrollers and IoT gadgets. Furthermore, supply chain attacks in 2026 are increasingly targeting every layer of the tech stack, from tampered software to compromised hardware and breached service providers, necessitating rigorous supply chain traceability for critical hardware and continuous monitoring of supplier environments.

Exploit Development

deep

Exploit development has seen significant activity leading up to July 18, 2026, with a record number of vulnerabilities addressed in Microsoft's July Patch Tuesday, totaling 622 fixes. This includes two zero-day vulnerabilities actively exploited in the wild: CVE-2026-56155, an elevation of privilege flaw in Active Directory Federation Services, and CVE-2026-56164, a SharePoint Server elevation of privilege vulnerability. Both issues were critical and could lead to administrative control or elevated privileges over a network, respectively. Additionally, a publicly disclosed BitLocker security feature bypass, CVE-2026-50661, was patched, which could allow physical attackers to access encrypted data. Several critical remote code execution (RCE) vulnerabilities in Windows DHCP Server, CVE-2026-50518 and CVE-2026-56159, both stemming from heap-based buffer overflows, were also resolved.

In other significant news, a 15-year-old Linux kernel flaw, GhostLock (CVE-2026-43499), was disclosed by Nebula Security, allowing local privilege escalation to root through a use-after-free vulnerability in the futex priority-inheritance code. This vulnerability, discovered by an AI agent, affects nearly all mainstream Linux distributions since 2011. Another critical KVM guest-to-host escape flaw, CVE-2026-46316 dubbed ITScape, affecting ARM64 KVM hosts was publicly disclosed with a proof-of-concept, exploiting a race condition in the vGIC-ITS emulation using a double-put primitive to achieve host kernel code execution. Separately, a critical memory corruption vulnerability, CVE-2026-44747, was identified in SAP NetWeaver Application Server ABAP, potentially leading to unauthorized data access or system unavailability.
