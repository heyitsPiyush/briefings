As of June 8, 2026, the exploit development landscape features several critical vulnerabilities under active exploitation and new techniques for memory corruption. Google patched an actively exploited Android zero-day, CVE-2025-48595, a privilege escalation flaw in the Android Framework affecting versions 14, 15, 16, and 16 QPR2, which does not require user interaction. Additionally, a Windows 11 kernel flaw, CVE-2026-40369, was disclosed with public exploit code, allowing attackers to escape Chrome, Edge, and Firefox sandboxes to gain SYSTEM access via an arbitrary 12-byte kernel write primitive in nt!ExpGetProcessInformation.

Further notable developments include an HTTP/2 "Bomb" exploit, CVE-2026-49975, capable of crashing major web servers like NGINX, Apache, IIS, Envoy, and Cloudflare Pingora by abusing HPACK compression and flow control; patches are available for NGINX and Apache, but IIS and Pingora remain vulnerable. In Linux kernel news, CVE-2026-31431, a local privilege escalation vulnerability in the cryptographic subsystem, allows unprivileged users to corrupt page cache memory and escalate privileges to root, with public exploit code available and confirmed in-the-wild activity. Also, CVE-2022-0492, an improper authentication flaw in the Linux kernel's cgroups v1 release_agent feature, is being actively exploited for privilege escalation, particularly in containerized environments. The Zero Day Initiative (ZDI) and Project Zero have continued to publish advisories and research, with ZDI detailing numerous vulnerabilities, including several Microsoft Edge flaws discovered during Pwn2Own Berlin 2026.
