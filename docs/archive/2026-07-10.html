Hardware Hacking

deep

Recent embedded security news reveals several critical firmware vulnerabilities and ongoing concerns in IoT security. RunZero disclosed seven vulnerabilities in FatFs, a filesystem library widely used in firmware for security cameras, drones, industrial controllers, and hardware crypto wallets. These flaws, including CVE-2026-6682, an integer overflow in FAT32 mount code, allow attackers to corrupt memory and execute arbitrary code, particularly when physical access enables the use of malicious USB drives or SD cards. Furthermore, the CERT Coordination Center revealed an unpatched firmware backdoor, CVE-2026-11405, affecting Tenda networking devices like the FH1201, W15E, AC10, AC5, and AC6 routers, allowing administrative access without credentials. Ubiquiti also released patches for seven critical UniFi OS vulnerabilities, including CVE-2026-50746, a command injection flaw in the UniFi Connect Application affecting building automation systems.

Side-channel and fault injection attacks remain a persistent threat, with Ledger Donjon researchers uncovering a laser fault injection vulnerability in Tangem crypto wallets that can reset passwords on all cards, for which no firmware fix is available. In IoT security research, the International Conference on IoT Security and Privacy (ICIoTSP - 26) is being held in Nagoya, Japan, in mid-July 2026, focusing on advancements in machine learning and big data analytics for IoT security. The FCC is also addressing supply chain security by proposing tighter approval processes for radiofrequency devices, including requirements for hardware and software bills of materials (HBOMs/SBOMs) to identify component origins. New hardware hacking tools include the M5Stack Cardputer, running Poseidon firmware, which offers 163 features for Wi-Fi, Bluetooth Low Energy, and Sub-GHz radio attacks, and the M5 Monster C5, built on the ESP32C5 chip, supporting dual-band Wi-Fi 6 for advanced Wi-Fi reconnaissance and attacks.
